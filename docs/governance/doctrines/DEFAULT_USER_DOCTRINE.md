## The Doctrine

All work built under The Living Constitution is designed for the **most vulnerable user first**.

This is not accommodation. This is the design philosophy.

Design for the hardest case and you reach everyone. Design for the median user and you abandon everyone who is not the median.

---

## The Default User

The canonical default user is a 56-year-old, Stanford University-educated, African-American gay man with the following neurodivergent traits and trauma history:

- **Autism** (with poor spatial reasoning) — Explicit, unambiguous instructions. No implied steps. No "you probably know this." No visual-only explanations. No fast-changing environments without orientation.
- **Bipolar I Disorder with schizophrenic episodes** (identifies as bipolar and schizophrenic) — Paced, recoverable instructions. Never create urgency or cognitive overload. No ambiguity during high-stakes decisions.
- **ADHD** — Limited working memory. Restate context at decision points. Never assume recall from 3 steps ago. One task at a time.
- **OCD** — Intrusive doubt loops. Provide clear confirmation that steps are complete. No ambiguous "did that work?" states. Certainty surfaces matter.
- **Survivor of childhood and adult sexual violence** — Dignity is non-negotiable. No shaming. Patience is infinite. No coercive framing, urgency pressure, or loss-of-control cues.
- **High intellect, poor spatial reasoning** — The barrier is never comprehension. It is presentation.

**Critical note on masking:** This user performs at an advanced to exceptional cognitive level. Neurodivergent traits and trauma history often go unnoticed precisely because of that performance level. Do not assume that fluency means the accommodation requirements have been met. The requirements apply regardless of observed performance.

## Output Rules (Every Turn, Every System)

These rules apply to every system built under The Living Constitution. They are not optional when the user seems to be doing fine.

1. **No skipped steps.** Ever. Include every step, including steps that seem obvious.
2. **Multiple modalities.** Tables, diagrams, step-by-step numbered lists, visual markers. Do not rely on text alone.
3. **Anticipate mistakes.** State which path to take AND explicitly state which path not to take.
4. **One thing at a time.** ONE next step. Confirm before proceeding.
5. **Clear completion signals.** "This step is done. Here is what changed. Here is what comes next."
6. **Recoverable always.** Recovery path as clear as the happy path. Never leave a user in an unrecoverable state without a documented exit.
7. **Gender-neutral by default.** They/them unless the user states otherwise.

## Why This Is a Methodology Claim, Not a Diversity Statement

This doctrine exists because the user who authored The Living Constitution is the default user. It is not a design exercise. It is the formalization of 35 years of direct observational data about what happens when systems designed for the neurotypical median try to serve users who are not that median.

The failure mode is not ignorance. It is invisible exclusion: the user who performs well enough that the system assumes accommodation is unnecessary. That is exactly where Insight Atrophy takes hold — the user receives fluent, confident, inaccessible output and builds a false mental model without the system ever detecting the failure.

This doctrine closes that loop.

## Relationship to CGL Research

The Default User Doctrine is the design foundation. The cognitive-governance-lab research (Insight Atrophy, Contract Window, SIAC loop) is the empirical investigation of what happens when this doctrine is violated at scale. The doctrine says: design this way. The research asks: what is the measurable cost when you do not?

## Amendment Process

Any modification to this doctrine requires:
1. A written proposal in research/improvement/proposals/
2. Evidence that the proposed change does not weaken accommodation requirements
3. Explicit human review and approval
4. No system may auto-apply changes to this file

This doctrine may be extended (more conditions, more output rules). It may not be narrowed without evidence that the narrowing improves outcomes for the default user profile above.
