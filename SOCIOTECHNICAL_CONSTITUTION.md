# Sociotechnical Constitution

This file is a repository-level orientation surface for the sociotechnical governance story around TLC. It is descriptive only. It is **not** the binding constitutional source for the repository and it is **not** a substitute for the executed project constitution at `projects/sociotechnical-constitution/SOCIOTECHNICAL_CONSTITUTION.md`.

## Binding sources

- Repository constitutional authority: `THE_LIVING_CONSTITUTION.md`
- Project-specific sociotechnical constitution instance: `projects/sociotechnical-constitution/SOCIOTECHNICAL_CONSTITUTION.md`
- Current operational truth: `STATUS.json`

## Sociotechnical framing

TLC treats governance as a relationship between:

- people making claims,
- AI systems producing drafts or actions,
- automated tooling enforcing or checking constraints,
- evidence surfaces that preserve what was actually done.

The repository therefore separates:

1. **Canonical rules** that govern what is allowed,
2. **Narrative explanations** that help operators understand structure,
3. **Derived summaries** that help users orient themselves without elevating those summaries into evidence.

## What this file is for

- Clarify that sociotechnical governance includes humans, AI assistants, and automated systems.
- Route readers to the binding and evidentiary surfaces.
- State what this repository is trying to support without overstating maturity.

## What is not claimed

- No claim of accessibility compliance certification.
- No claim that sociotechnical effectiveness has been empirically validated here.
